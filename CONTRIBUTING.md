# Contributing to Container Security Series

Thank you for your interest in contributing! 🎉

## How to Contribute

### Reporting Issues

- **Inaccurate information?** Open an issue with the specific section and correction.
- **Broken links?** Open an issue with the link and suggested replacement.
- **Missing topic?** Open an issue describing what should be added.

### Pull Requests

1. Fork the repository
2. Create a branch (`git checkout -b fix/your-fix`)
3. Make your changes in the `docs/` folder
4. Commit with a clear message (`git commit -m "fix: correct Trivy command in Part 2"`)
5. Push and open a PR

### Content Guidelines

- Use **Vietnamese + English technical terms** (matching existing style)
- Include **runnable code examples** where possible
- Cite sources for statistics and claims
- Keep YAML/code examples copy-paste ready
- Test commands before submitting

### What We're Looking For

- 🐛 Bug fixes (typos, incorrect commands, broken YAML)
- 📊 Updated statistics (newer reports, surveys)
- 🛠️ New tool examples or comparisons
- 🔒 New CVEs or attack techniques
- 🌐 Translations

## Code of Conduct

Be respectful. Be constructive. Focus on the content.
